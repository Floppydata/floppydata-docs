## Regular browsers

### Chrome / Edge

<Tabs>
  <Tab title="Windows">
    **Option 1: Using system proxy settings**

    1. Open Windows Settings → **Network & Internet** → **Proxy**
    2. Under "Manual proxy setup", toggle **On**
    3. Enter proxy details:
       - **Address**: `geo.g-w.info`
       - **Port**: `10080`
    4. Click **Save**

    **Option 2: Using Chrome extension**

    See [Browser Extensions](#browser-extensions) section below
  </Tab>
  <Tab title="macOS">
    1. Open **System Preferences** → **Network**
    2. Select your active connection (Wi-Fi or Ethernet)
    3. Click **Advanced** → **Proxies** tab
    4. Check **Web Proxy (HTTP)** and/or **Secure Web Proxy (HTTPS)**
    5. Enter:
       - **Server**: `geo.g-w.info`
       - **Port**: `10080`
       - **Username**: Your Floppydata username
       - **Password**: Your Floppydata password
    6. Click **OK** → **Apply**
  </Tab>
</Tabs>

<Warning>
  **Authentication in Chrome/Edge system proxy:** Windows/Mac system proxies don't always prompt for authentication. If you get "407 Proxy Authentication Required" errors, use a browser extension instead (see below).
</Warning>

### Firefox

Firefox has built-in proxy settings with authentication support:

<Steps>
  <Step title="Open Settings">
    Menu (≡) → **Settings** → **General**

    Scroll down to **Network Settings** → Click **Settings...**
  </Step>
  <Step title="Configure proxy">
    Select **Manual proxy configuration**

    Enter:

    - **HTTP Proxy**: `geo.g-w.info`
    - **Port**: `10080`
    - Check **"Also use this proxy for HTTPS"**
  </Step>
  <Step title="Test connection">
    Click **OK** and visit [http://ip-api.com](http://ip-api.com)

    Firefox will prompt for username and password — enter your Floppydata credentials
  </Step>
</Steps>

<Tip>
  **Smart Parameters in Firefox:** When Firefox prompts for username, you can add Smart Parameters:

  ```text
  user123-res-country-US-session-firefox01
  ```
</Tip>

## Antidetect browsers

Antidetect browsers are designed for multi-accounting and managing multiple browser profiles with different fingerprints and proxies.

### Dolphin Anty

**Popular for:** E-commerce, social media, traffic arbitrage

<Steps>
  <Step title="Create new profile">
    Click **"New Profile"** in Dolphin Anty
  </Step>
  <Step title="Configure proxy">
    In the profile creation window:

    1. Go to **Proxy** section
    2. Select proxy type: **HTTP** or **SOCKS5**
    3. Enter:
       - **Host**: `geo.g-w.info`
       - **Port**: `10080` (HTTP) or `10800` (SOCKS5)
       - **Login**: `user123-res-country-US-session-profile01`
       - **Password**: Your password
    4. Click **Check Proxy** to test
  </Step>
  <Step title="Save and launch">
    Click **"Create"** and launch the profile

    Each profile can have different proxy settings!
  </Step>
</Steps>

<Tip>
  **Pro tip:** Use unique session IDs for each profile:

  - Profile 1: `user123-res-country-US-session-profile01`
  - Profile 2: `user123-res-country-US-session-profile02` This gives each profile a different IP address.
</Tip>

## Browser extensions

Extensions make it easy to switch proxies on/off without changing system settings.

### FoxyProxy (Chrome, Firefox, Edge)

**Most popular proxy extension**

<Steps>
  <Step title="Install extension">
    - [Chrome Web Store](https://chrome.google.com/webstore/detail/foxyproxy)
    - [Firefox Add-ons](https://addons.mozilla.org/firefox/addon/foxyproxy-standard/)
  </Step>
  <Step title="Add proxy">
    1. Click FoxyProxy icon → **Options**
    2. Click **Add** (new proxy)
    3. Enter:
       - **Title**: Floppydata US
       - **Type**: HTTP or SOCKS5
       - **Hostname**: `geo.g-w.info`
       - **Port**: `10080` or `10800`
       - **Username**: `user123-res-country-US`
       - **Password**: Your password
    4. Click **Save**
  </Step>
  <Step title="Activate proxy">
    Click FoxyProxy icon → Select your proxy

    The icon will turn colored when proxy is active
  </Step>
</Steps>

<Tip>
  **Create multiple proxies** for different locations:

  - Floppydata US: `user123-res-country-US`
  - Floppydata UK: `user123-res-country-GB`
  - Floppydata Mobile: `user123-mobile-country-US`
</Tip>

  - WebRTC not leaking real IP
</Check>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Proxy not connecting" icon="plug">
    **Check:**

    1. Credentials are correct (copy fresh from dashboard)
    2. Port is correct: `10080` (HTTP) or `10800` (SOCKS5)
    3. Host is exactly: `geo.g-w.info` (not .com)
    4. No extra spaces in username/password

    **Test:**

    ```bash
    curl -x http://USERNAME:PASSWORD@geo.g-w.info:10080 http://ip-api.com/json
    ```
  </Accordion>

  <Accordion title="407 Authentication error" icon="key">
    **Causes:**

    - Wrong username or password
    - Special characters not encoded
    - Extra spaces

    **Solutions:**

    1. Copy credentials again from dashboard
    2. Remove any spaces before/after
    3. Try resetting password in dashboard
  </Accordion>

  <Accordion title="Slow connection" icon="gauge">
    **Try:**

    1. Switch proxy type: Datacenter is fastest
       - `user123-dc-country-US`
    2. Choose closer location to your target
    3. Check if target website is slow (not proxy issue)
  </Accordion>

  <Accordion title="IP keeps changing" icon="arrows-rotate">
    **Cause:** You're not using a session parameter

    **Solution:** Add session to keep same IP:

    ```text
    user123-res-country-US-session-stable01
    ```
  </Accordion>

  <Accordion title="Website detects proxy" icon="eye">
    **Try:**

    1. Use Mobile proxies (highest trust)
       - `user123-mobile-country-US`
    2. Use residential instead of datacenter
    3. Add realistic user agent in browser
    4. Consider Web Unlocker for heavily protected sites
  </Accordion>
</AccordionGroup>

## Best practices

<CardGroup cols={2}>
  <Card title="For multi-accounting" icon="users">
    - Use separate session ID per account
    - Never share proxies between linked accounts
    - Use Static ISP for important accounts (Facebook ads, Amazon)
    - Stay consistent (don't change countries)
  </Card>

  <Card title="For privacy" icon="lock">
    - Use residential proxies (look like real users)
    - Enable rotation for different IP each time
    - Check for DNS/WebRTC leaks
    - Use HTTPS websites
  </Card>

  <Card title="For social media" icon="heart">
    - Mobile proxies have highest trust
    - Use unique session per account
    - Don't switch locations frequently
    - Add realistic delays between actions
  </Card>

  <Card title="For e-commerce" icon="cart-shopping">
    - Static ISP proxies prevent account linking
    - One dedicated IP per store/account
    - Never log into multiple accounts from same IP
    - Keep proxy consistent over time
  </Card>
</CardGroup>
