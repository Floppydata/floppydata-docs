---
title: "How Floppydata Works"
---

Most proxy services work like a catalogue: you choose an IP, connect to it, and use it. Floppydata works differently.

You don’t select individual proxies. Instead, you describe how your request should behave – and Floppydata handles the routing automatically.

### **The core idea**

When you connect through Floppydata, you pass instructions along with your credentials.

These instructions – called **Smart Parameters** – define:

- where the request should originate from (country, city)
- which IP type should be used (residential, mobile, datacenter)
- whether the IP should rotate or stay the same (session)
- how requests should be distributed

Floppydata reads these parameters and routes your request through the appropriate network.

**You never:**

- manage proxy lists
- check if an IP is alive
- rotate IPs manually
