<Note>
  Floppydata offers **five proxy types** grouped into **two categories**:

  - **Rotating proxies** – IPs are assigned from a shared pool and can change across requests
  - **Static proxies** – a fixed IP is assigned to your account

  Each type differs in trust level, speed, cost, and ideal use cases.
</Note>

## Rotating proxies

Rotating proxies assign IPs from a shared pool of real network connections. Each request is routed through a different IP unless you control it via sessions.

You don’t get a permanent IP. Floppydata automatically selects and rotates IPs based on your parameters

<Note>
  Rotating proxies are billed based on **traffic (GB)**, not individual IPs.
</Note>

- Residential — real user IPs with high trust
- Mobile — mobile carrier IPs with the highest trust
- Datacenter — fast and cost-efficient IPs for scale

## Static proxies

Static proxies provide a fixed IP assigned to your account.

They are best suited for workflows where IP consistency matters more than rotation — for example, account management, long sessions, or tools that require the same trusted IP over time.

<Note>
  Static proxies are billed **per IP per month**.
</Note>

- Static ISP
- Static Datacenter

## How to choose

Use the table below to find the right proxy type for your workflow:

| Use case | Recommended proxy | Why |
| --- | --- | --- |
| Social media accounts (Facebook, TikTok, LinkedIn) | Mobile or Static ISP | High trust required, stable identity |
| Ad accounts (Meta Ads, Google Ads) | Static ISP or Mobile | Geo-consistency and account stability |
| E-commerce (Amazon, eBay, Etsy) | Static ISP | Fixed IP required to avoid account linking |
| Marketplaces / seller accounts | Static ISP | Rotation may trigger bans |
| Payment platforms (PayPal, Stripe, banking) | Static ISP | IP changes trigger security checks |
| Crypto / KYC platforms | Static ISP | Requires consistent identity |
| Account creation (social media) | Mobile | Mobile IPs have highest trust |
| Multi-accounting | Mobile or Static ISP | Separate identity per account |
| SEO & SERP tracking | Residential | Accurate geo-targeting |
| Price monitoring | Residential | Access localized content |
| Ad verification | Residential or Mobile | Real-user IP simulation |
| Web scraping (protected sites) | Residential | Higher success rate |
| Web scraping (high volume) | Datacenter | Faster and cheaper at scale |
| Web scraping (balanced) | Residential | Balance between trust and scale |
| Automation tools / bots | Datacenter or Static Datacenter | Speed and performance |
| API access / integrations | Static Datacenter | Stable IP for allowlisting |
| QA / geo-testing | Residential | Simulate real user locations |
| Sneaker / ticketing bots | Mobile or Residential | Avoid detection systems |
| Betting platforms | Mobile | Highest trust level required |

<Note>
  • Use **Rotating proxies** when you need scale and request distribution\
  • Use **Static proxies** when you need a stable identity
</Note>
